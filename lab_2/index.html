Assignment 1

What I have learned

Basic concepts of Responsive Web Design

What is the purpose of RWD?

Answer: RWD can be used to design web pages that can be compatible (different in layout, fontsize, etc.) with different screen sizes

Why it is used?

Answer: This era is digital era. More and more people are using mobile devices. Different kind of input devices are introduced, etc. In order to deliver the same user experience for different type of devices (desktop, mobile). RWD is needed to tailor the specific needs for these types of devices.

How things are done before RWD?

Answer: Before RWD, pages need to have different versions to target different kind of devices, which probably can be time-consuming tasks.

What is the difference?

Answer: Developers can use the same pages and using CSS to target different type of media screens, so that they can ensure that pages are going to look good and usable on different devices

How css media queries can help you building responsive web sites?

Answer: CSS media queries combines screen types and other physical characteristics such as min width of the screen, max width of the screen

Why to use em(font) based sizing for example margins?

Answer: Whenever we want to build a responsive website, which means that it can be scaled according to different screens, we need to use "em". "Em" unit will relative to the font size and will scale elements according to font size. So the proportional can be the same for different screens no matter what.

Article is already 7 years old, do we have any new technology which can be used to achieve same type of UI responsiveness as described in the article. Buzz words (CSS Grid, Flexbox)

Answer: There are 2 new features that can be used to design responsive web-sites which are CSS Flexbox and CSS Grid. By using those techniques, developers dont need to use that much media queries and can still make responsive web-sites in short time

After you have done the basic layout, how could you prevent user zooming the site on mobile

Answer: By adding viewport meta tag: <meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">
